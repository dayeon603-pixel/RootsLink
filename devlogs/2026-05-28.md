### Results / Observations
- All 117 tests use `SimpleNamespace` duck-typing — no database connection required.
  Algorithm functions only access attributes (.interests, .skills, .barriers, .country,
  etc.); Python duck-typing allows any object with the right attributes.
- `backend.database` creates a SQLite engine at import time; SQLAlchemy engine creation
  is lazy (no file I/O) so importing works without a `rootslink.db` file present.
- Timing tests use `date.today() + timedelta(days=N)` — date-independent.
- rootslink algorithm test suite: 117 tests (was 0, +117).

### Hypotheses
- H-duck1: SQLAlchemy ORM model classes can be replaced by `SimpleNamespace` for pure
  algorithm unit tests — Python ignores type annotations at runtime. Confirmed by design.
